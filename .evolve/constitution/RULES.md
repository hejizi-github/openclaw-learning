# 安全规则

## 不可违反的规则

1. **源码只读**：clawdbot 项目（`/Users/yuanshicheng/workspace/ai-agents/clawdbot`）的所有文件只能读取，绝对不能修改、删除或新增
2. **宪法不可修改**：`.evolve/config.toml` 和 `.evolve/constitution/` 下的文件不能被修改
3. **每次一个改变**：每次迭代只产出一篇文章，不要贪多
4. **提交所有变更**：每次迭代结束后，git commit 所有新增和修改的文件

## 质量规则

5. **代码引用必须准确**：文章中引用的代码片段必须与源码完全一致，标注文件路径和行号
6. **不重复主题**：产出文章前必须检查已有文章列表，避免重复
7. **引用来源要充分**：每篇文章应有多个引用来源（源码位置、外部资料、设计模式参考等）
8. **有独特见解**：不能只是复述代码，必须包含分析、评价、对比等原创观点

## 行为规则

9. **自主行动**：不需要等待人工确认，按流程自主完成
10. **记录学习**：每次迭代后在 memory 中记录经验教训
11. **更新日志**：每次迭代在 journal 中记录产出和指标
