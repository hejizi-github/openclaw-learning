# 工作流：clawdbot 源码教学内容生产

## 概述

每次迭代产出一篇关于 clawdbot 项目的教学文章。Agent 自主探索源码，选择有价值的主题，结合联网调研，产出有深度和独特见解的 Markdown 文章。

## 流程步骤

### 1. 选题（Explore）
- 阅读 clawdbot 项目源码（`/Users/yuanshicheng/workspace/ai-agents/clawdbot`）
- 检查已有文章，避免重复主题
- 从架构设计、核心模块、设计模式、工程实践等维度发现值得讲解的主题
- 选择一个有深度可挖的主题

### 2. 深度研究（Research）
- 深入阅读该主题相关的所有源码文件
- 理解实现细节、设计决策、取舍权衡
- 联网搜索相关的技术背景、行业实践、类似项目对比
- 收集足够多的引用来源

### 3. 撰写（Write）
- 产出一篇 Markdown 教学文章
- 要求：
  - 有深度：不是表面描述，要剖析"为什么这样设计"
  - 有见解：提出自己的观点和评价，不只是复述代码
  - 引用充分：源码引用标注文件路径和行号，外部资料附链接
  - 教学导向：读者能从中学到可迁移的知识
- 文章放在项目根目录下，文件名用有意义的英文命名

### 4. 自检（Verify）
- 检查所有代码引用是否与源码一致
- 确认没有与已有文章重复的主题
- 统计引用来源数量，记录到 journal

### 5. 反思（Reflect）
- 回顾本次产出的质量
- 记录学到的经验（什么主题值得深挖、什么写法效果好）
- 更新 memory 和 journal
